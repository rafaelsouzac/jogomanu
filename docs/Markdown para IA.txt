# Contexto

# Regras de Negócio (`Business Rules`)
- **Autenticação**: Entrada via CPF (11 dígitos) e senha numérica (6 dígitos).
- **Segurança**: 
  - Bloquear conta por `15 minutos` após `3 tentativas inválidas`.
  - Token de recuperação via SMS para "Esqueci a Senha".

# Entregáveis Solicitados
1. **Histórias de Usuário** (Padrão: *Como um... Eu quero... Para...*)
2. **Critérios de Aceitação** (Padrão Gherkin: *Dado que... Quando... Então...*)
3. **Fluxo de Exceção** (Cenário de bloqueio por tentativas)

# Dicionário de Dados

## Tabela: `usuarios`

| Campo | Tipo | Restrição | Descrição |
| :--- | :--- | :--- | :--- |
| `id` | INT | **PK**, AUTO_INCREMENT | Identificador único do usuário |
| `nome` | VARCHAR(100) | NOT NULL | Nome completo |
| `email` | VARCHAR(100) | NOT NULL, UNIQUE | E-mail de login |

## Tabela: `pedidos`

| Campo | Tipo | Restrição | Descrição / Relacionamento |
| :--- | :--- | :--- | :--- |
| `id` | INT | **PK**, AUTO_INCREMENT | Identificador único do pedido |
| `usuario_id` | INT | **FK** (referencia `usuarios.id`) | Cliente que realizou o pedido |
| `data_pedido` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | Data e hora da compra |
| `total` | DECIMAL(10,2) | NOT NULL | Valor total do pedido |
